<h1 align="center">AntiSpamBot</h1>

<p align="center">
  <i>Подозрительно косится на каждого нового участника.</i>
</p>

Telegram анти-спам бот на Go. При входе нового участника в группу показывает капчу с 6 цветными кружочками — пользователь должен выбрать кружок указанного цвета за 30 секунд. Неправильный ответ или таймаут → кик (с возможностью перезайти). Три провала подряд → перманентный бан.

Дополнительно собирает статистику: сколько зашло, прошли ли капчу, активность новичков против старичков, топ писателей, топ провалов. Команда `/stats` в чате показывает сводку за сутки/неделю/месяц/всё время. Плюс ловит «молчаливых возвращенцев» — если кто-то не писал 30+ дней (настраивается) и вдруг ожил, бот в чате отвечает «Ого! X молчал Y и вот написал».

Работает в Docker, разворачивается на VDS одной командой. Состояние хранится в SQLite (один файл) — рестарт бота не теряет незавершённые капчи и статистику.

## Как это выглядит

**При входе нового участника** бот отправляет сообщение:

> **Привет, @vasya!**
> Для защиты от спама выбери **красный** кружок за 30 секунд.
> [🟢] [🟡] [🔴] [🔵] [🟣] [🟠]

- Нажал 🔴 → ограничения сняты, сообщение удалено, добро пожаловать.
- Нажал другой / не нажал за 30с → кикается, сообщение удаляется.
- После 3 провалов подряд за 24ч — перманентный бан.

**По команде `/stats`** (только для админов):

```
📊 Статистика за неделю

👋 Новых участников: 47
• Прошли капчу: 31 (66%)
• Кикнуты: 14 (29%)
• Забанены: 2 (4%)

💬 Сообщений: 1,234
• Новички: 89 (7%)
• Старички: 1,145 (93%)

🔝 Топ писателей:
1. Alice — 340 сообщений
2. @bob — 287 сообщений
3. Charlie — 194 сообщения
...

🚫 Топ провалов капчи:
1. @spam_bot_1 — 3 раза
2. id9999 — 2 раза

Новичок — тот, кто прошёл капчу за последние 7 дн.
```

**Ещё бот ловит «молчаливых возвращенцев»:**

> 🎊 Сенсация! **Vasya** молчал(а) **2 года** и вот наконец-то написал(а)!

Триггерится автоматически, когда пользователь пишет в чате после долгой паузы (≥ `SILENT_ANNOUNCE_DAYS`, по умолчанию 30). Тексты разные в зависимости от длительности: день/месяц/год.

## Возможности

- Капча с цветными кружками на inline-кнопках
- Настраиваемый таймаут и порог бана
- **Персистентность:** SQLite. Рестарт не ломает незавершённые капчи, счётчики провалов и статистика сохраняются.
- **Статистика:** заходы/прохождения/кики/баны + активность новичков vs старичков. Команда `/stats [day|week|month|all]`.
- Работа в нескольких чатах одновременно (с опциональным whitelist'ом)
- Команды `/start` и `/help` в личке
- Структурные JSON-логи (`slog`), уровни `debug|info|warn|error`
- Graceful shutdown по `SIGINT`/`SIGTERM`
- Multi-stage Docker build → итоговый образ ~15 МБ на Alpine, запуск от non-root пользователя

## Быстрый старт

### 1. Создание бота

1. В Telegram напиши [@BotFather](https://t.me/BotFather) → `/newbot`, придумай имя и username.
2. Сохрани полученный **BOT_TOKEN** вида `123456789:ABC-def...`.
3. У BotFather выключи **Privacy Mode**: `/mybots → <your_bot> → Bot Settings → Group Privacy → Turn off`. Без этого бот не увидит события присоединения и сообщения.

### 2. Локальный запуск

Нужен Go 1.25+.

```bash
git clone https://github.com/menand/AntiSpamBot.git
cd AntiSpamBot
cp .env.example .env
# впиши BOT_TOKEN в .env
go mod tidy
go run ./cmd/bot
```

Бот автоматически подхватит переменные из `.env` (через `godotenv`). SQLite-файл создастся в текущей папке (`./bot.db` по умолчанию; настраивается через `DB_PATH`).

### 3. Деплой на VDS через Docker

На сервере нужен только Docker с плагином Compose (Ubuntu/Debian):

```bash
# если Docker ещё не установлен:
curl -fsSL https://get.docker.com | sh

git clone https://github.com/menand/AntiSpamBot.git
cd AntiSpamBot
cp .env.example .env
nano .env   # впиши BOT_TOKEN
docker compose up -d --build
```

SQLite-файл живёт в Docker volume `bot-data`, который **переживает** `docker compose down/up --build`. Volume удалится только при `docker compose down -v`.

Проверить логи:

```bash
docker compose logs -f
```

Остановить:

```bash
docker compose down
```

Обновить до свежей версии (данные не потеряются):

```bash
git pull && docker compose up -d --build
```

Автостарт при перезагрузке VDS обеспечен `restart: unless-stopped` в `docker-compose.yml`.

## Добавление бота в чат

1. Открой нужную группу → «Управление группой» → «Администраторы» → «Добавить администратора» → найди своего бота.
2. **Обязательные права админа:**
   - ✅ **Блокировать пользователей** (Ban users) — для кика и бана
   - ✅ **Удалять сообщения** (Delete messages) — чтобы убирать сообщение с капчей
3. Остальные права можно выключить.

> ⚠️ **Важно:** без статуса админа Telegram **не присылает** события `chat_member`, и бот вообще не узнаёт о новых участниках.

Если используешь `ALLOWED_CHATS` — узнай ID своего чата: перешли любое сообщение из группы в [@userinfobot](https://t.me/userinfobot), он покажет ID вида `-1001234567890`.

## Конфигурация

Все настройки — через переменные окружения (`.env` файл при Docker compose).

| Переменная                | По умолчанию              | Описание                                                |
|---------------------------|---------------------------|---------------------------------------------------------|
| `BOT_TOKEN`               | **обязат.**               | Токен бота от @BotFather                                |
| `CAPTCHA_TIMEOUT_SECONDS` | `30`                      | Сколько секунд даётся на прохождение капчи              |
| `MAX_ATTEMPTS`            | `3`                       | После скольких провалов подряд — перманентный бан       |
| `NEWCOMER_DAYS`           | `7`                       | Сколько дней после прохождения капчи считается новичком |
| `SILENT_ANNOUNCE_DAYS`    | `30`                      | Порог «молчания» для авто-анонса возвращенца. `0` — выкл. |
| `LOG_LEVEL`               | `info`                    | `debug` \| `info` \| `warn` \| `error`                  |
| `DB_PATH`                 | `bot.db` (в Docker: `/data/bot.db`) | Путь к SQLite-файлу                          |
| `ALLOWED_CHATS`           | (не задана)               | Chat ID через запятую. Если задан — работает только в них |
| `OWNER_IDS`               | (не задана)               | Telegram user_id владельцев через запятую. Получают уведомления о старте/остановке в личку и могут запускать `/stats` без проверки «админ ли ты в этом чате». |

## Команды бота

Все команды видны в «/»-подсказке в Telegram (бот регистрирует их при старте через `setMyCommands`):

- `/start`, `/help` — **в личке**. Показывает меню с inline-кнопками («Справка», «Добавить в группу», и для владельцев — «Мои чаты»). Также показывает твой Telegram ID (удобно для настройки `OWNER_IDS`).
- `/stats [day|week|month|all]` — **в группе**, для админов чата (или для владельцев бота, указанных в `OWNER_IDS`). Выводит статистику за период. Без аргумента — за неделю. Русские алиасы: `/stats сутки|неделя|месяц|все`.
- `/chats` — **в личке**, только для владельцев. Список всех чатов, где работает бот, с inline-кнопками для просмотра статистики каждого (с переключателями периода прямо в DM).

## Настройка владельца

Чтобы быть «супер-админом» бота (получать уведомления о старте, видеть `/stats` без проверки прав в каждом чате):

1. В Telegram напиши своему боту `/start` — он ответит и покажет твой Telegram ID.
2. Впиши его в `.env`:
   ```
   OWNER_IDS=123456789
   ```
   Можно через запятую несколько: `OWNER_IDS=123,456`.
3. Перезапусти бота (`docker compose up -d --build`). После старта бот пришлёт тебе в личку сообщение «🟢 Бот запущен».

> ⚠️ Если после настройки `OWNER_IDS` ты **не получил** сообщение о старте — проверь, что ты хотя бы раз писал боту в личку. Telegram не даёт ботам инициировать переписку.

## Статистика

### Что собирается

| Событие | Когда записывается |
|---|---|
| `join`  | Пользователь зашёл в чат, бот начал капчу |
| `pass`  | Пользователь прошёл капчу правильно |
| `kick`  | Провалил капчу (ошибка или таймаут) — кикнут |
| `ban`   | Провалил капчу 3+ раз подряд за 24ч — перманентный бан |

Плюс:
- Агрегаты по дням: сколько сообщений в сумме написали новички vs старички.
- Per-user статистика: сколько сообщений каждый написал (`user_message_counts` по дням), когда было первое и последнее сообщение (`user_activity`), кэш отображаемого имени/username (`user_info`).

### Что НЕ собирается

- **Содержимое сообщений** — только факт, время и автор.
- **Ничего за пределами чатов, где бот работает.**

Сырые данные лежат в SQLite-файле (`/data/bot.db` в Docker). При желании можно подключаться к нему через `sqlite3` и писать произвольные запросы.

> ⚠️ **Приватность.** Бот хранит per-user статистику сообщений (кто когда писал, сколько). Если разворачиваешь для чата, где пользователи это не ожидают — предупреди их или выставь `SILENT_ANNOUNCE_DAYS=0` и не используй `/stats`. Можно периодически чистить данные вручную через `sqlite3` (см. раздел «Очистка статистики» ниже).

## Разработка

```bash
make build         # go build
make run           # go run
make test          # go test -race ./...
make vet           # go vet ./...
make docker-up     # docker compose up -d --build
make docker-logs   # docker compose logs -f
make docker-down   # docker compose down
```

Основные зависимости:
- [`github.com/mymmrac/telego`](https://github.com/mymmrac/telego) — Telegram Bot API клиент (Bot API 7.x+)

CI (GitHub Actions): `vet` + `test -race` + `build` на каждом push/PR.

## Операционные вопросы

**Логи.** Пишутся в stdout в JSON (`slog`). Docker compose настроен на ротацию: максимум 3 файла по 10 МБ.

**Рестарт.** `restart: unless-stopped` — контейнер сам поднимется после падения или ребута VDS.

**Обновления.** `git pull && docker compose up -d --build`. SQLite-файл в volume сохраняется.

**Бэкапы.** SQLite-файл `/data/bot.db` в volume `bot-data`. Забэкапить:

```bash
docker compose exec bot sqlite3 /data/bot.db ".backup /data/bot.db.bak"
# скопировать на хост:
docker cp menand-antispam:/data/bot.db.bak ./backups/
```

**Миграции схемы.** Схема применяется при старте (`CREATE TABLE IF NOT EXISTS`). При добавлении новых таблиц/колонок достаточно дописать `ALTER TABLE` / `CREATE TABLE IF NOT EXISTS` в `internal/storage/schema.sql`.

**Очистка статистики.** Можно зайти в SQLite и удалить старые записи вручную:

```bash
docker compose exec bot sqlite3 /data/bot.db
> DELETE FROM events WHERE at < strftime('%s', 'now', '-1 year');
> DELETE FROM message_counts WHERE day < date('now', '-1 year');
> DELETE FROM user_message_counts WHERE day < date('now', '-1 year');
> -- Полный сброс per-user статистики:
> DELETE FROM user_activity;
> DELETE FROM user_message_counts;
> DELETE FROM user_info;
```

## Траблшутинг

**Бот молчит при входе новых пользователей.**
- Проверь, что бот — **админ** с правами «Банить» и «Удалять сообщения».
- У BotFather отключи **Privacy Mode**.
- Посмотри логи: `docker compose logs --tail=50`. Если там тихо — Telegram не шлёт апдейты, дело в правах.

**`/stats` отвечает «команда доступна только администраторам».**
Бот проверяет статус вызывающего через `getChatMember`. Проверь, что ты действительно админ чата (создатель тоже считается).

**Ошибка `CHAT_ADMIN_REQUIRED`.**
Бот попытался кикнуть / ограничить пользователя без прав. Проверь админские права бота.

**`BOT_TOKEN is not set`.**
Нет `.env` файла или в нём пустой `BOT_TOKEN=`. `cp .env.example .env` и впиши токен.

**`database is locked` в логах.**
Редкая ситуация при очень высокой нагрузке. Бот настроен на `busy_timeout=5s` и один writer — должно хватать. Если видишь систематически — значит чат очень активный, перейди на Postgres/MySQL (потребуется переписать `internal/storage`).

**Пользователь прошёл капчу, но не может писать.**
`release()` не сработал (нет прав или сетевая ошибка). Сними ограничения вручную в настройках чата. В логах будет `release` с err.

**После рестарта контейнера — капчи работают?**
Да, это главная причина добавления SQLite. Бот при старте читает `pending_captchas`, перезапускает таймеры с правильным оставшимся временем, и если срок уже истёк — сразу выполняет кик.
