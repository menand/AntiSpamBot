# AntiSpamBot

Telegram анти-спам бот на Go. При входе нового участника показывает капчу с цветными кружками — пользователь должен выбрать кружок указанного цвета. Неудача → кик (с возможностью перезайти), 3 неудачи → перманентный бан.

## Что делает

- Слушает события `chat_member` во всех чатах, где бот — админ.
- На каждого нового живого пользователя (не-бота):
  1. Ограничивает все права в чате.
  2. Отправляет капчу: «Выбери <цвет> кружок» + 6 кнопок (🔴🟢🔵🟡🟣🟠 в случайном порядке).
  3. Ждёт ответ 30 секунд.
- Правильный ответ → снимает ограничения, удаляет сообщение с капчей.
- Неправильный / таймаут → кик + удаление сообщения. На 3-й раз — перманентный бан.
- Ботов (`user.IsBot == true`), добавленных админом, пропускает.

## Требования к боту в чате

Чтобы бот работал, он должен быть **администратором** с правами:
- **Ban users** (для кика/бана)
- **Delete messages** (для удаления капчи)

Без статуса админа Telegram не шлёт updates `chat_member`, и бот вообще не узнает о новых участниках.

## Запуск локально

```bash
cp .env.example .env
# положите в .env свой BOT_TOKEN от @BotFather
go mod tidy
go run ./cmd/bot
```

## Запуск на VDS через Docker

```bash
git clone <repo-url>
cd MenandAntiSpam
cp .env.example .env
# отредактируйте .env
docker compose up -d --build
docker compose logs -f
```

Остановить: `docker compose down`. Обновить: `git pull && docker compose up -d --build`.

## Конфигурация

| Переменная                | По умолчанию | Описание                                  |
|---------------------------|--------------|-------------------------------------------|
| `BOT_TOKEN`               | (обязательно) | Токен бота от @BotFather                  |
| `CAPTCHA_TIMEOUT_SECONDS` | `30`         | Сколько секунд на прохождение капчи       |
| `MAX_ATTEMPTS`            | `3`          | После скольких провалов — перманентный бан |

## Структура проекта

```
cmd/bot/             — main (entrypoint)
internal/config/     — парсинг env-переменных
internal/captcha/    — генерация challenge + in-memory store pending-капч
internal/attempts/   — счётчик провалов на (chatID, userID) с TTL
internal/bot/        — telego-клиент, хендлеры, действия (restrict/ban/kick)
```

## Известные ограничения (MVP)

- Состояние капч и счётчиков — **в памяти**. После рестарта контейнера незавершённые капчи теряются: пользователи, которых бот успел ограничить, останутся с ограничениями до следующего админского действия. Счётчики попыток тоже обнуляются.
- Нет персистентности — если нужно пережить рестарты, можно добавить SQLite/Redis.
- Один инстанс. Горизонтальное масштабирование не поддерживается (in-memory store).
